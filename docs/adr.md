# Architecture Decision Records

|         | Nome                  | Racional                                                                                                                                                                                                                                                                      |
| ------- | --------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| ADR-001 | FIX 4.4               | Embora o FIX 5.0+ apresente muitas melhorias de performance, foi selecionado o FIX 4.4 pois<br />é amplamente mais adotado até a presente data e isso facilita as integrações.                                                                                         |
| ADR-002 | App Client mínimo    | O app Client tem uma estrutura de pastas e design pattern mínimo para funcionar como um<br />"tiny stateless proxy" para uma melhor performance                                                                                                                              |
| ADR-003 | Validações de Dados | As validações dos dados são feitas apenas na camada de negócios da aplicação para diminuir o<br />overhead de transformações e checagens nos dados transitados                                                                                                       |
| ADR-004 | Generated Ids         | Ids são gerados utilizando UUID v7, para permitir ordenação nativa pelo timestamp embutido                                                                                                                                                                                 |
| ADR-005 | Result struct         | Utilizaçao de struct Result para normalizar o retorno da camada de negócios, permitindo o<br />encapsulamento de Exceptions, e transformando o tratamento de erros para o nível de negócio da<br />aplicação, tendo assim uma melhor performance                       |
| ADR-006 | Hard Locks            | Uso de locks para evitar data-racing em concorrência nos repositórios em memoria.<br />Considerando que as operações são muito pequenas e rápidas e o fluxo é sequencial, esta tecnica <br />permite uma baixa perda de latencia sem necessidade de maior complexidade |
| ADR-007 | Implementação FIX   | Implementação dos services e apps FIX baseada no repositório oficial de exemplos:<br />https://github.com/connamara/quickfixn/tree/master/Examples                                                                                                                         |
| ADR-008 | Paralelismo e async   | Apenas foi utilizado async tasking nas requisicoes Http. Para as requisicoes FIX, não faz sentido<br />o uso de async/parallel pois se trata de um canal tcp sempre aberto e requisições senquenciais                                                                      |
